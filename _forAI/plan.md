## 미결 항목

- **`/mcp reconnect all` 이 실패한 서버를 세션 재시작 없이 되살리는가.** 미검증.
  `/mcp` 출력이 그렇게 권하지만 실제로 확인하지 않았다. `readme.md` 는 이 방법을 싣지 않았다.
- **설정 파일을 세션 도중에 고쳤을 때 자동 반영되는가.** 미검증.
  2026-07-09 관측은 전부 "재시작 후 붙었다" 로 설명된다. 자동 감지의 증거는 없다.
  가르는 실험: `/mcp` 를 건드리지 않고 서버를 하나 더 등록한 뒤 도구가 저절로 나타나는지 본다.
- **비-editable(wheel) 설치에서의 `profiles.yaml` 경로.** 미검증.
  `APP_DIR = Path(__file__).resolve().parents[2]`는 src 레이아웃 editable 설치를 전제한다.
  site-packages 설치 시 어디로 떨어지는지, 쓰기가 되는지 확인하지 않았다. 현재 사용 방식
  (`uv run gui-tool`)에서는 문제가 없다.

## Structure decisions

- **Claude Code는 `.mcp.json` (project scope) 하나만 사용한다.** `-s user`와 `-s local` 등록은
  2026-07-09 제거했다. user scope는 모든 프로젝트를 오염시킨다.
- **Codex는 사용자 전역 MCP 등록을 사용한다.** VS Code에서 상위 폴더를 작업 루트로 열어도
  Chrome DevTools MCP를 쓰기 위한 2026-07-14 결정이다. 프로젝트 `.codex/config.toml`은
  저장소 루트 사용을 위한 예시로 유지한다.
- **`.claude/` 는 커밋하지 않는다.** 승인 기록이며, 커밋하면 클론한 사람의 승인 관문이 사라진다.
- `gui_tool/`은 독립 uv 프로젝트이며 PyYAML만 외부 Python 의존성으로 사용한다.
  잔존 프로세스 회수도 stdlib ctypes로 구현했다 — psutil을 넣지 않는다.
- SSH 연결·Chrome 실행·프로파일 저장·GUI는 모두 Python으로 처리한다. PowerShell 스크립트는 없다.
- **앱이 소유한 Chrome과 ssh만 Job Object로 종료하고, 기존 Chrome은 건드리지 않는다.**
  이 규칙은 잔존물 정리 기능에도 그대로 적용된다 — 소유권 증거(세션 파일의 pid+이미지+생성
  시각, 또는 전용 프로파일 폴더 점유)가 확인된 것만 종료한다.
- **UI 프레임워크는 Tkinter를 유지한다.** 2026-07-22에 PySide6 전환을 검토했으나, 실측으로
  확인된 결함 12건 중 Tk가 원인인 것은 이벤트 펌프 관련 2건뿐이었고 둘 다 소규모 수정으로
  해결됐다. 나머지는 전부 Win32 프로세스 관리와 도메인 로직이라 프레임워크를 바꿔도 그대로
  남는다. 동시 터널 다중 관리나 실시간 프로세스 테이블이 필요해지면 재검토한다.
- **정리 기능은 자동 실행하지 않는다.** 시작 시 검사는 하되, 남은 **프로세스**가 있을 때만
  확인을 묻는다. 폴더만 남은 경우는 로그에만 적는다 — 매 실행마다 묻는 것은 성가시고, 전용
  프로파일에는 로그인 상태가 들어 있을 수 있다.
- **AI 협업은 GUI 내장 JSON-RPC + 얇은 MCP 브리지로 한다.** 2026-07-22 결정이며, 앞으로 만드는
  GUI 협업 툴의 표준 구조다. 헤드리스 모드는 만들지 않는다 — 목적이 AI에게 별도 인스턴스를
  주는 것이 아니라 사람이 보는 창을 함께 조작하게 하는 것이기 때문이다. 상세는 `memo.md`의
  "GUI 협업 툴 구조" 절.
- **AI 경로에서는 포트를 자동 전환하지 않는다.** `chrome-devtools-mcp`의 `--browser-url`은 MCP
  서버 시작 시점에 고정되므로, 포트가 바뀌면 이미 붙어 있는 연결이 끊긴다. 대화형 복구는 사람이
  창에서 확인할 때만 동작한다.

## Risks

- **`@latest` 는 조용히 올라간다.** `readme.md` 에 박은 "1.5.0 / 도구 29개" 는 언젠가 틀려진다.
- **로그인 세션 노출.** `--autoConnect` 로 내 창에 붙으면 열린 탭·쿠키·인증 헤더가 에이전트 컨텍스트로 들어간다.
  `--redactNetworkHeaders` 기본값은 `false` 다. 이 저장소는 autoConnect 를 다루지 않는다.
- **프롬프트 인젝션.** 에이전트가 읽는 페이지가 에이전트에게 명령할 수 있다.
- **DevTools 포트 노출.** 원격 `9222`는 반드시 `127.0.0.1`에만 바인딩한다.
- **GUI의 비대화.** SSH 비밀번호 저장, 서비스 설치, 자동 재연결은 현재 범위에 넣지 않는다.
- **실제 reverse forward 미검증.** SSH 공개키 인증과 명령 생성은 확인했지만, 원격 서버의
  `curl http://127.0.0.1:9222/json/version`은 실행 환경에서 아직 확인하지 않았다.
- **잔존물 정리의 오탐 위험.** pid는 Windows에서 빠르게 재사용된다. 생성 시각 대조를 빼면
  남의 프로세스를 죽인다. 종료 직전 재확인 로직을 제거하지 말 것.
- **미검증 항목을 검증된 것처럼 쓰지 말 것.** 위 "미결 항목" 세 개는 전부 미검증이다.
